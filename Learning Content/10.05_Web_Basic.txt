1. Web Basic Outline
 - Web = 가게(회사)
 - 사장 = 서버
   - 백엔드 = 데이터베이스 + 사용자제어
     - source : java파일(서블릿), jsp(서블릿으로 변환)
       - 컴파일 필요
     - resource : properties / xml (정보)
 - 손님 = 클라이언트
   - 프론트엔드 = 디자인 + 퍼블리싱 + ...
     - resource = 이미지, html/html5, css, javascript/jquery
       - 컴파일 필요 없음!
 - 싱글톤(singleton)
   - 미리 개수를 정해서 만들어서 관리 = 한정수량 판매
   - 장점 : 효율성, 관리, 속도
   - 단점 : 불특정 상황에 사용 불가
 - 프로토타입(prototype)
   - 원할 때 생성(new) = 주문 제작
   - 장점 : 불특정 상황에 대처 가능
   - 단점 : 속도 느림, 관리가 어려움
 - 스프링은 통제, 제어가 잘 되는 쪽을 선호
   - 특별한 말이 없으면 무조건 싱글톤 사용
 - jsp에서는 개발자가 선택해서 사용
   - 특별한 말이 없으면 프로토타입으로 생각
